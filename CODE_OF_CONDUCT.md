# Procedural Terrain Toolkit Code of Conduct

## Our Commitment

Procedural Terrain Toolkit is an open-source project built for the Unity community. We want this repository to be a welcoming, technically rigorous, and respectful space for everyone who contributes, asks questions, reports bugs, or shares ideas.

We are committed to maintaining a community where collaboration is professional, constructive, and safe regardless of age, body size, disability, ethnicity, gender identity or expression, experience level, education, nationality, personal appearance, race, religion, sexual identity or orientation, or technology preferences.

## Expected Behavior

Community members are expected to:

- be respectful in discussions, reviews, and issue reports
- assume good intent while still giving clear technical feedback
- focus criticism on code, design decisions, performance trade-offs, and documentation rather than on people
- welcome new contributors and help keep project guidance understandable
- use evidence when discussing performance claims, regressions, frame spikes, memory usage, or GPU/CPU behavior
- respect maintainer decisions around repository scope, release timing, and architectural direction

## Unacceptable Behavior

The following behavior is not acceptable in this community:

- harassment, intimidation, bullying, or personal attacks
- discriminatory or hateful language
- sexualized language or unwanted sexual attention
- trolling, baiting, or deliberately inflammatory behavior
- repeated bad-faith arguments after a maintainer has asked for de-escalation
- doxxing, stalking, threats, or publishing private information without consent
- spam, malicious links, or intentionally disruptive issue/PR activity
- using project spaces to insult, shame, or target contributors

## Scope

This Code of Conduct applies to all repository-managed spaces, including:

- issues
- pull requests
- discussions
- code review threads
- release comments
- other public or semi-public spaces tied to this repository

It also applies when an individual is representing the project in community spaces, technical discussions, or public collaboration channels related to the toolkit.

## Reporting Concerns

If you experience or witness behavior that violates this Code of Conduct:

1. **For non-sensitive problems**, open a discussion or issue if public context is appropriate.
2. **For sensitive or personal concerns**, contact the repository owner privately through GitHub profile contact options rather than posting details publicly.
3. If the behavior also violates GitHub platform rules, use GitHub's abuse and reporting tools as appropriate.

When making a report, please include:

- what happened
- where it happened
- links or screenshots if available
- whether the situation is ongoing

We will review reports in good faith and aim to respond in a fair, consistent, and respectful manner.

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct. They may take any action they deem appropriate in response to unacceptable behavior, including:

- requesting changes in tone or behavior
- editing or removing comments
- closing or locking conversations
- rejecting contributions
- temporarily restricting participation
- permanently banning repeat or severe offenders

Enforcement decisions will be based on the severity of the behavior, the pattern of conduct, community safety, and the health of the project.

## Conflict Resolution Expectations

Technical disagreement is normal in a performance-focused project. Contributors are encouraged to:

- bring profiler captures, benchmarks, traces, or reproduction cases when possible
- separate architecture debate from personal criticism
- step back when a thread stops being productive
- let maintainers make the final call when a decision is needed to move the project forward
